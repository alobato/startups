= PUBLISHING THE GEM

#first make sure rubyforge is configured locally. taken from here: http://lazybuddha.berkeley.edu/display/istas/How+to+Publish+a+Gem+on+Rubyforge#HowtoPublishaGemonRubyforge-ConfigureRubyforgeLocally
rubyforge setup
rubyforge config
rubyforge create_package startups startups

#had to manually add processor ids after getting this error:  no <processor_id> configured for <Any>
vi ~/.rubyforge/auto-config.yml
processor_ids: 
  IA64: 6000
  AMD-64: 1500
  Any: 8000
  Sparc: 4000
  PPC: 2000
  Other: 9999
  Alpha: 7000
  i386: 1000
  UltraSparc: 5000
  MIPS: 3000


#prepare the gem for installation
rake manifest

#install the gem locally
rake install

#publish to rubyforge (packages and uploads the gem) (make sure to change the version number and checkin first)
rake release

#publish the docs
rake publish_docs

#uninstall the gem locally
rake uninstall

#see if it's available after release:
gem list --remote startups

IMPORT THE PROJECT TO THE SVN REPOSITORY
-------------------
from INSIDE the project dir, do:
svn import -m "importing startups project" . svn+ssh://ryanowens@rubyforge.org/var/svn/startups/trunk

#NOTE: svn did not seem to like me using ~ instead of Users/ryan
#NOTE: if you need to delete a project (for example if you forgot /trunk like I did), do
       svn delete -m "Deleting project dir" svn+ssh://ryanowens@rubyforge.org/var/svn/startups

#then do the whole copy to tmp dir, and checkout back to the same place, so...
mv startups startups_tmp
svn co svn+ssh://ryanowens@rubyforge.org/var/svn/startups/trunk startups
